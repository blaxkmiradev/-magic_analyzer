## What each section tells you

**File Type Detection** вҖ” Matches 70+ magic byte signatures at their correct offsets. Drills into ZIP containers to distinguish `.docx` / `.xlsx` / `.pptx` / APK. Breaks down RIFF containers into WAV, AVI, or WebP.

**Format-specific Headers** вҖ” Goes beyond just the type. For example:
- **PNG** вҶ’ width, height, color type, bit depth, interlace mode
- **ELF** вҶ’ architecture (x86-64, ARM, RISC-VвҖҰ), endianness, OS ABI, binary type
- **PE/EXE** вҶ’ target machine, compile timestamp, DLL vs executable, subsystem
- **SQLite** вҶ’ page size, WAL mode, encoding, page count
- **GZIP** вҶ’ original filename, OS, timestamp, compression level
- **ZIP** вҶ’ compression method, first entry name, version needed
- **JPEG** вҶ’ dimensions, DPI/density, EXIF presence
- **PDF** вҶ’ version, encryption, XFA forms

**Entropy** вҖ” Shannon entropy of the first 512 bytes. Useful for spotting encrypted payloads, compressed blobs, or suspiciously random data hiding inside innocent-looking files.

**Extension Mismatch** вҖ” Warns when the file's actual content doesn't match its extension. Catches renamed files, obfuscated malware, or just accidental mis-saves.

**Hashes** вҖ” MD5, SHA-1, SHA-256 of the full file for integrity checks and VirusTotal lookups.
